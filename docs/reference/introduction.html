You are about to kick off the development of your super digital banking application and cannot wait to start writing those lines of code. Here are the key steps you'll need to follow to integrate with our Digital Banking Enabler API.

Authentication

You have to get authenticated to use the API — this means you need an auth token to make your request. There are two different types of users: the BIB Users and Enterprise Users.

BIB User

To get your Auth token as another BIB user follow the steps below:

1
Contact the Business or Implementation team
Reach out to initiate your integration request and confirm your use case and environment requirements.
2
Sign SOW & provide your authorized token custodian
Complete the Statement of Work and designate a token custodian within your organization.
3
Staging environment setup
Your staging environment will be provided for use by the implementation support team.
4
Join the Implementation support team via group chat
A dedicated group chat is created for your integration to provide real-time support.

Enterprise User

To get your auth token as an enterprise user follow the steps below:

  • Log into iRestrict
  • Set up your account
  • Get your authentication token
  • Join Teams group provided by the Implementation support team
Authorization Header 
Authorization: Bearer <your_auth_token>

// Example request with auth token
fetch('https://api.hexcore.io/v2/accounts', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${authToken}`,
    'Content-Type': 'application/json'
  }
})

Testing Your Actions

After successfully getting your auth token and setting up your account, you might want to confirm your API actions. For instance, after creating an account via API, user can view this account on BankOne to confirm the account was created as required.

To do this, login to BankOne using the Institution code: 4-digit institution code, Username: Username and Password provided by the implementation team.

ℹ️
Always test your integrations thoroughly in the Staging environment before going live. Never use production credentials in your development environment.

API Environment

hexCore provides two environments for your integration journey:

NameDescription
Production This is the environment where real action and transactions occur and should be used when you are ready to put your application into operation for the intended users. An auth-token would be provided for this environment following sign-off of staging test.
Staging This is the environment where you are required to conduct your test. A staging auth token would be generated for your institution and should be used for this purpose.

What you should Know about Our API

The hexCore API is categorized into two main types namely:

  • Corebanking APIs — Direct integration with the core banking system for account management, transactions, and product operations.
  • Channels APIs — Integration with customer-facing channels for digital banking interactions.
⚠️
The Corebanking & Channels API have different base URLs; hence it is important to take note of this difference when integrating or conducting your test.
Base URLs 
# Corebanking API
Production: https://api.hexcore.io/v2
Staging:    https://staging-api.hexcore.io/v2

# Channels API
Production: https://channels.hexcore.io/v2
Staging:    https://staging-channels.hexcore.io/v2

Error Codes

The hexCore API uses standard HTTP response codes. Here are the most common ones:

CodeMeaningAction
200SuccessRequest completed successfully.
201CreatedResource created successfully.
400Bad RequestCheck your request body — a required field may be missing or malformed.
401UnauthorizedYour auth token is missing, expired, or invalid. Refresh your token.
403ForbiddenYour account does not have permission to perform this action.
404Not FoundThe requested resource does not exist.
429Too Many RequestsYou have exceeded your rate limit. Implement exponential backoff.
500Server ErrorAn unexpected error on our end. Contact support if this persists.
